# tanstack-demo — Agent Context

## Scaffolding history

Created with the TanStack CLI:

```
npx @tanstack/cli@latest create my-tanstack-app --agent \
  --add-ons tanstack-query,neon,better-auth,drizzle,form,sentry
```

Followed by:

```
npx @tanstack/intent@latest install
npx @tanstack/intent@latest list
```

Package manager: **pnpm**. Template: blank React (TanStack Start). Public GitHub repo: `echoja/tanstack-demo`.

## Stack

- TanStack Start + Router + Query + Form + Devtools
- Better Auth (email + password, TanStack Start cookies plugin, Drizzle adapter)
- Drizzle ORM + drizzle-kit
- PostgreSQL via `pg`
- Sentry (`@sentry/tanstackstart-react`, `instrument.server.mjs`)
- Tailwind v4

## Environment variables

See `.env.example`. Loaded via `dotenv-cli` in the `dev` script; provided via `env_file: .env.local` in `docker-compose.yml`.

- `DATABASE_URL` — Postgres connection string. Local dev example: `postgres://tanstack_demo:tanstack_demo@localhost:5432/tanstack_demo`. Kubernetes uses the in-cluster Postgres service.
- `BETTER_AUTH_URL` — public origin of the app (`https://tanstack-demo.echoja.com` in prod, `http://localhost:37291` in dev).
- `BETTER_AUTH_SECRET` — random 32-byte hex. Generated once and kept in `.env.local`.
- `VITE_SENTRY_DSN`, `VITE_SENTRY_ORG`, `VITE_SENTRY_PROJECT`, `SENTRY_AUTH_TOKEN` — optional; app runs without Sentry if blank.

## Deployment (home hosting)

- **Port**: `37291` (hard-coded in `package.json` dev script and `docker-compose.yml`; published only on `127.0.0.1`).
- **Container**: `docker compose up -d --build`. The container runs `pnpm db:migrate` before starting the server, so schema changes apply on boot.
- **Database**: PostgreSQL runs separately in Kubernetes as a StatefulSet with a PVC. The app is stateless apart from external DB access.
- **Public hostname**: `tanstack-demo.echoja.com`, terminated by the existing Cloudflare Tunnel (`~/.cloudflared/config.yml`, tunnel id `2521553e-c23b-4708-9625-50a9fd65fb49`) and routed into k3d on `localhost:18081`.
- **Preview hostname pattern**: `pr-<number>.tanstack-demo.echoja.com`, created by Argo CD's pull request ApplicationSet.
- After editing `~/.cloudflared/config.yml`, send `kill -HUP <pid>` to the running `cloudflared` to reload (launchd auto-restarts it).

## Key architectural decisions

- Postgres schema lives in `src/db/schema.ts` and includes both app tables (`todos`) and Better Auth's core tables (`user`, `session`, `account`, `verification`). Single drizzle schema -> single `drizzle-kit` workflow for everything.
- `src/db/index.ts` creates a `pg` pool and exports a typed Drizzle client.
- Better Auth is wired via `drizzleAdapter(db, { provider: 'pg', schema: {...} })` in `src/lib/auth.ts`.
- No Neon vite plugin — removed `neon-vite-plugin.ts` and `db/init.sql` (drizzle migrations replace the seed).
- The scaffold's `src/routes/demo/neon.tsx` and the Header link to it were removed.

## Scripts

- `pnpm dev` — vite dev server on port 37291 with Sentry instrumentation.
- `pnpm db:generate` / `db:migrate` / `db:push` / `db:studio` — drizzle-kit workflows.
- `pnpm build` / `pnpm start` — production build + start (used inside the container).

## Known gotchas

- The app requires `DATABASE_URL`; without it the server fails fast during startup.
- The TanStack Router route tree (`src/routeTree.gen.ts`) is generated by the vite plugin on first run; `tsc --noEmit` on a fresh clone will report missing module until you run `pnpm dev` or `pnpm build` once.
- `docker compose up` binds to `127.0.0.1:37291` only. Do not expose publicly; the cloudflared tunnel is the intended ingress.
- TanStack Start's build target here emits only a `dist/server/server.js` fetch-handler module — no standalone HTTP listener. A small Node adapter, `server-entry.mjs` at the repo root, wraps it with `node:http` and terminates on `$HOST:$PORT`. Start script: `node --import ./dist/server/instrument.server.mjs server-entry.mjs`.
- `server-entry.mjs` serves `/health` directly for Kubernetes probes.
- `docker-compose.yml` uses `env_file: .env.local` for secrets. Do **not** also interpolate the same var in `environment:` (e.g. `BETTER_AUTH_SECRET: ${BETTER_AUTH_SECRET}`) — compose interpolates from the host shell and will silently override env_file with an empty string.
